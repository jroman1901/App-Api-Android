Crear una API REST con Node.js y MySQL

Guia paso a paso para construir una API de gestion de tareas

0

Que vamos a construir

Vamos a crear una API REST completa con Node.js, Express y MySQL que permita gestionar tareas (CRUD).

Metodo Ruta Descripcion
GET/tareasObtener todas las tareas
GET/tareas/:idObtener una tarea por ID
POST/tareasCrear una nueva tarea
PUT/tareas/:idActualizar una tarea
DELETE/tareas/:idEliminar una tarea
1

Requisitos previos

Antes de comenzar, asegurate de tener instalado lo siguiente en tu computadora:

  • Node.js — version 18 o superior. Descargalo en nodejs.org
  • MySQL — version 8. Puedes usar XAMPP o MySQL Workbench
  • Un editor de codigo — recomendamos Visual Studio Code

Verifica que Node.js esta instalado abriendo una terminal y ejecutando:

node --version
npm --version

Si ambos comandos muestran un numero de version, estas listo para continuar.

2

Crear la base de datos en MySQL

Abre MySQL Workbench o tu cliente de MySQL y ejecuta el siguiente SQL para crear la base de datos y la tabla de tareas:

⚠ El codigo debe escribirse a mano — la copia esta desactivada
script.sql 
CREATE DATABASE db_tareas;

USE db_tareas;

CREATE TABLE tareas (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  titulo      VARCHAR(100) NOT NULL,
  descripcion TEXT,
  completada  BOOLEAN DEFAULT false
);
La columna id se incrementa automaticamente. completada empieza en false por defecto.
3

Inicializar el proyecto Node.js

Crea una carpeta para tu proyecto y abrela en la terminal:

mkdir api-tareas
cd api-tareas

Inicializa el proyecto con npm para generar el archivo package.json:

npm init -y

Ahora instala las dependencias que necesitamos:

npm install express mysql2 dotenv
  • express — framework para crear el servidor HTTP
  • mysql2 — driver para conectarse a MySQL
  • dotenv — para leer variables de entorno desde un archivo .env
4

Crear el archivo .env

Crea un archivo llamado .env en la raiz del proyecto. Este archivo guarda las credenciales de la base de datos sin exponerlas en el codigo.

⚠ El codigo debe escribirse a mano — la copia esta desactivada
.env 
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=
DB_NAME=db_tareas
Nunca subas el archivo .env a GitHub. Agrega .env a tu archivo .gitignore.
5

Crear la conexion a la base de datos — db.js

Crea el archivo db.js en la raiz del proyecto. Este modulo se encarga unicamente de conectarse a MySQL y exportar la conexion para usarla en otras partes del proyecto.

⚠ El codigo debe escribirse a mano — la copia esta desactivada
db.js 
const mysql = require('mysql2');
require('dotenv').config();

const connection = mysql.createConnection({
  host:     process.env.DB_HOST     || 'localhost',
  user:     process.env.DB_USER     || 'root',
  password: process.env.DB_PASSWORD || '',
  database: process.env.DB_NAME     || 'db_tareas'
});

connection.connect((err) => {
  if (err) {
    console.error('Error conectando a MySQL:', err);
    return;
  }
  console.log('Conectado a MySQL correctamente');
});

module.exports = connection;
require('dotenv').config() carga las variables del archivo .env en process.env para que podamos usarlas de forma segura.
6

Crear el servidor y las rutas — index.js

Crea el archivo index.js. Este es el archivo principal. Aqui configuramos Express y definimos todos los endpoints de la API.

Primero: importa las dependencias y crea la app

⚠ El codigo debe escribirse a mano — la copia esta desactivada
index.js — Configuracion inicial 
const express = require('express');
const db      = require('./db');

const app = express();
app.use(express.json());
express.json() permite que el servidor entienda el cuerpo de las peticiones en formato JSON (necesario para POST y PUT).

Segundo: ruta GET para obtener todas las tareas

index.js — GET /tareas 
// GET - Obtener todas las tareas
app.get('/tareas', (req, res) => {
  db.query('SELECT * FROM tareas', (err, results) => {
    if (err) return res.status(500).json({ error: err.message });
    res.json(results);
  });
});

Tercero: ruta GET para obtener una tarea por ID

index.js — GET /tareas/:id 
// GET - Obtener una tarea por ID
app.get('/tareas/:id', (req, res) => {
  db.query(
    'SELECT * FROM tareas WHERE id = ?',
    [req.params.id],
    (err, results) => {
      if (err) return res.status(500).json({ error: err.message });
      if (results.length === 0)
        return res.status(404).json({ mensaje: 'Tarea no encontrada' });
      res.json(results[0]);
    }
  );
});
El ? en la query SQL es un parametro seguro (prepared statement). Evita inyecciones SQL. El valor se pasa en el arreglo [req.params.id].

Cuarto: ruta POST para crear una tarea

index.js — POST /tareas 
// POST - Crear una tarea
app.post('/tareas', (req, res) => {
  const { titulo, descripcion } = req.body;
  db.query(
    'INSERT INTO tareas (titulo, descripcion) VALUES (?, ?)',
    [titulo, descripcion],
    (err, result) => {
      if (err) return res.status(500).json({ error: err.message });
      res.status(201).json({ mensaje: 'Tarea creada', id: result.insertId });
    }
  );
});

Quinto: ruta PUT para actualizar una tarea

index.js — PUT /tareas/:id 
// PUT - Actualizar una tarea
app.put('/tareas/:id', (req, res) => {
  const { titulo, descripcion, completada } = req.body;
  db.query(
    'UPDATE tareas SET titulo = ?, descripcion = ?, completada = ? WHERE id = ?',
    [titulo, descripcion, completada, req.params.id],
    (err, result) => {
      if (err) return res.status(500).json({ error: err.message });
      if (result.affectedRows === 0)
        return res.status(404).json({ mensaje: 'Tarea no encontrada' });
      res.json({ mensaje: 'Tarea actualizada' });
    }
  );
});

Sexto: ruta DELETE para eliminar una tarea

index.js — DELETE /tareas/:id 
// DELETE - Eliminar una tarea
app.delete('/tareas/:id', (req, res) => {
  db.query(
    'DELETE FROM tareas WHERE id = ?',
    [req.params.id],
    (err, result) => {
      if (err) return res.status(500).json({ error: err.message });
      if (result.affectedRows === 0)
        return res.status(404).json({ mensaje: 'Tarea no encontrada' });
      res.json({ mensaje: 'Tarea eliminada' });
    }
  );
});

Septimo: iniciar el servidor al final del archivo

index.js — Iniciar servidor 
const PORT = 3000;
app.listen(PORT, () => {
  console.log(`Servidor corriendo en http://localhost:${PORT}`);
});
7

Estructura final del proyecto

Al terminar, tu proyecto debe tener esta estructura de archivos:

estructura del proyecto 
api-tareas/
├── node_modules/
├── .env
├── db.js
├── index.js
└── package.json
8

Correr el servidor y probarlo

Inicia el servidor desde la terminal:

npm start

Deberias ver en consola:

salida esperada 
Conectado a MySQL correctamente
Servidor corriendo en http://localhost:3000

Ahora puedes probar la API. Puedes usar Postman, Thunder Client (extension de VS Code) o curl.

Ejemplo — crear una tarea con curl:

curl -X POST http://localhost:3000/tareas -H "Content-Type: application/json" -d "{\"titulo\":\"Mi primera tarea\",\"descripcion\":\"Descripcion de prueba\"}"

Ejemplo — obtener todas las tareas:

curl http://localhost:3000/tareas
Si usas Postman, importa las rutas manualmente. Recuerda siempre agregar el header Content-Type: application/json en las peticiones POST y PUT.
