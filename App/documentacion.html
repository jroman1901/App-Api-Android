Resumen del Proyecto

Una app Android nativa que consume una API REST CRUD de tareas alojada en un servidor local.

🎯 Funcionalidades

  • Listar todas las tareas desde la API
  • Crear nuevas tareas (titulo + descripcion)
  • Marcar tareas como completadas (toggle)
  • Eliminar tareas con un solo toque
  • Indicador de carga (spinner) durante peticiones
  • Manejo de errores con Toast

🔧 Stack Tecnologico

  • Lenguaje: Kotlin 2.0.21
  • UI: Jetpack Compose + Material 3
  • HTTP: Retrofit 2.11.0
  • Cliente HTTP: OkHttp3 4.12.0
  • JSON: Gson (Retrofit converter)
  • Concurrencia: Kotlin Coroutines

Paso 1 — Modelo de Datos: Tarea.kt

Define la estructura JSON que la API envia y recibe, mapeada a una data class Kotlin.

Archivo clave

Tarea.kt — Data class

Representa una tarea tal como la API la devuelve. Usa @SerializedName para mapear el campo JSON completada (Int 0/1 de MySQL) a una propiedad Kotlin.

package com.umg.appapi

import com.google.gson.annotations.SerializedName

data class Tarea(
    val id: Int? = null,             // null al CREAR (lo asigna la BD)
    val titulo: String,              // requerido
    val descripcion: String? = "",   // opcional
    @SerializedName("completada")    // nombre del campo en el JSON
    val completadaInt: Int = 0       // 0=pendiente, 1=completada (MySQL TINYINT)
) {
    // Propiedad calculada: convierte Int -> Boolean para la UI
    val completada: Boolean get() = completadaInt == 1
}
⚠

Por que Int y no Boolean? MySQL almacena booleanos como TINYINT(1) (0/1). Si se usa Boolean directamente, Gson puede fallar al deserializar dependiendo de la API. El campo completadaInt: Int es 100% seguro y compatible.

💡

@SerializedName indica a Gson que el campo JSON se llama "completada" pero en Kotlin lo guardamos como completadaInt para evitar conflictos con la propiedad calculada completada: Boolean.

Ejemplo de JSON de la API

{
  "id":          1,
  "titulo":       "Estudiar Kotlin",
  "descripcion": "Repasar coroutines",
  "completada":  0           // 0=false, 1=true
}

Paso 2 — Cliente HTTP: TareaApi.kt

Define los endpoints con anotaciones Retrofit y construye el cliente HTTP con OkHttp.

Archivo clave

TareaApi.kt — Interface + Factory

Retrofit lee las anotaciones (@GET, @POST, etc.) e implementa automaticamente la interfaz generando el codigo HTTP.

package com.umg.appapi

import okhttp3.OkHttpClient
import okhttp3.logging.HttpLoggingInterceptor
import retrofit2.Retrofit
import retrofit2.converter.gson.GsonConverterFactory
import retrofit2.http.*

interface TareaApi {

    @GET("tareas")
    suspend fun getTareas(): List<Tarea>

    @GET("tareas/{id}")
    suspend fun getTarea(@Path("id") id: Int): Tarea

    @POST("tareas")
    suspend fun createTarea(@Body tarea: Tarea): Tarea

    @PUT("tareas/{id}")
    suspend fun updateTarea(@Path("id") id: Int, @Body tarea: Tarea): Tarea

    @DELETE("tareas/{id}")
    suspend fun deleteTarea(@Path("id") id: Int)

    companion object {
        // IP especial del emulador para acceder al localhost del host
        private const val BASE_URL = "http://10.0.2.2:3000/"

        fun create(): TareaApi {
            // 1. Interceptor que loguea request y response en Logcat
            val logger = HttpLoggingInterceptor().apply {
                level = HttpLoggingInterceptor.Level.BODY
            }

            // 2. OkHttpClient con el interceptor adjunto
            val client = OkHttpClient.Builder()
                .addInterceptor(logger)
                .build()

            // 3. Retrofit con base URL, cliente y conversor JSON
            return Retrofit.Builder()
                .baseUrl(BASE_URL)
                .client(client)
                .addConverterFactory(GsonConverterFactory.create())
                .build()
                .create(TareaApi::class.java)  // implementacion automatica
        }
    }
}

Anotaciones Retrofit explicadas

AnotacionDonde se usaQue hace
@GET("tareas")FuncionPeticion GET a BASE_URL + "tareas"
@POST("tareas")FuncionPeticion POST con cuerpo JSON
@PUT("tareas/{id}")FuncionPeticion PUT, el {id} es un parametro de ruta
@DELETE("tareas/{id}")FuncionPeticion DELETE por ID
@Path("id")ParametroSustituye {id} en la URL con el valor real
@BodyParametroSerializa el objeto Kotlin a JSON y lo envia en el body
✅

suspend fun: Todas las funciones son suspend, lo que permite llamarlas dentro de una coroutine sin bloquear el hilo principal de Android (UI thread). Retrofit con Kotlin las ejecuta en un hilo I/O automaticamente.

Componente 2 de 3

TareaScreen — Pantalla principal

Gestiona el estado de la lista, el formulario y las operaciones CRUD. Es el corazon de la UI.

@Composable
fun TareaScreen(api: TareaApi, modifier: Modifier = Modifier) {
    val scope    = rememberCoroutineScope()   // scope vinculado al ciclo de vida
    val context  = LocalContext.current

    // --- ESTADO REACTIVO ---
    var tareas      by remember { mutableStateOf(emptyList<Tarea>()) }
    var titulo      by remember { mutableStateOf("") }
    var descripcion by remember { mutableStateOf("") }
    var isLoading   by remember { mutableStateOf(false) }

    // --- FUNCION LOCAL: carga tareas de la API ---
    fun cargarTareas() {
        scope.launch {
            isLoading = true
            try {
                tareas = api.getTareas()   // coroutine: no bloquea UI
            } catch (e: Exception) {
                Toast.makeText(context, "Error: ${e.message}", Toast.LENGTH_LONG).show()
            } finally {
                isLoading = false
            }
        }
    }

    // Carga automatica al montar el composable (equivale a onCreate)
    LaunchedEffect(Unit) { cargarTareas() }

    Column(modifier = modifier.padding(16.dp)) {
        // --- FORMULARIO NUEVA TAREA ---
        Text("Nueva Tarea", style = MaterialTheme.typography.headlineSmall)
        OutlinedTextField(value = titulo,      onValueChange = { titulo = it },      label = { Text("Titulo") })
        OutlinedTextField(value = descripcion, onValueChange = { descripcion = it }, label = { Text("Descripcion") })

        Button(onClick = {
            if (titulo.isBlank()) return@Button
            scope.launch {
                api.createTarea(Tarea(titulo = titulo, descripcion = descripcion))
                cargarTareas()       // refresca la lista
                titulo = ""; descripcion = ""
            }
        }) { Text("Agregar") }

        // --- LISTA DE TAREAS ---
        LazyColumn {
            items(tareas) { tarea ->
                TareaItem(
                    tarea    = tarea,
                    onDelete = { scope.launch { api.deleteTarea(tarea.id!!); cargarTareas() } },
                    onToggle = {
                        scope.launch {
                            val actualizada = tarea.copy(completadaInt = if (tarea.completada) 0 else 1)
                            api.updateTarea(tarea.id!!, actualizada)
                            cargarTareas()
                        }
                    }
                )
            }
        }
    }
}
Componente 3 de 3

TareaItem — Tarjeta individual

Composable "tonto" (stateless): solo recibe datos y callbacks. Muestra checkbox, titulo, descripcion e icono de borrar.

@Composable
fun TareaItem(tarea: Tarea, onDelete: () -> Unit, onToggle: () -> Unit) {
    Card(modifier = Modifier.fillMaxWidth().padding(vertical = 4.dp)) {
        Row(modifier = Modifier.padding(16.dp), verticalAlignment = Alignment.CenterVertically) {
            Checkbox(checked = tarea.completada, onCheckedChange = { onToggle() })
            Column(modifier = Modifier.weight(1f)) {
                Text(tarea.titulo,            style = MaterialTheme.typography.titleMedium)
                Text(tarea.descripcion ?: "", style = MaterialTheme.typography.bodySmall)
            }
            IconButton(onClick = onDelete) {
                Icon(Icons.Default.Delete, contentDescription = "Eliminar")
            }
        }
    }
}
💡

Patron Hoisting: TareaItem no tiene estado propio. El estado sube (hoists) al padre TareaScreen, haciendo el componente reutilizable y testeable. Cuando el usuario toca Checkbox o Delete, llama a los callbacks onToggle/onDelete.

Paso 4 — AndroidManifest.xml

Declara los permisos necesarios para conectarse a la red y permite trafico HTTP sin cifrar.

Archivo: AndroidManifest.xml

Permisos y configuracion de red

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <!-- Permite al app hacer peticiones HTTP -->
    <uses-permission android:name="android.permission.INTERNET" />

    <application
        android:usesCleartextTraffic="true"   <!-- Permite HTTP (no solo HTTPS) -->
        android:theme="@style/Theme.AppApi">

        <activity
            android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
        </activity>
    </application>
</manifest>
⚠

INTERNET: Sin este permiso la app lanza SecurityException al intentar cualquier peticion de red. Es obligatorio.

🔐

usesCleartextTraffic=true: Android 9+ bloquea HTTP por defecto. Como el backend corre en HTTP (no HTTPS) se necesita esta bandera. En produccion se debe usar HTTPS.

Flujo de Ejecucion Completo

Como interactuan todos los componentes desde que el usuario abre la app hasta ver la lista.

Carga inicial de tareas

  • 1
    App inicia — MainActivity.onCreate() es llamado. Crea una instancia de TareaApi via TareaApi.create(). Monta el tema y el Scaffold.
  • 2
    TareaScreen se compone — Jetpack Compose ejecuta la funcion. Se crea el estado inicial: lista vacia, campos en blanco, isLoading=false.
  • 3
    LaunchedEffect(Unit) — Al aparecer por primera vez en pantalla, Compose lanza automaticamente la coroutine cargarTareas().
  • 4
    isLoading = true — Compose re-renderiza y muestra el CircularProgressIndicator en la barra de titulo.
  • 5
    api.getTareas() — Retrofit construye la peticion GET http://10.0.2.2:3000/tareas. OkHttp la ejecuta en un hilo I/O. El hilo UI queda libre.
  • 6
    Respuesta JSON recibida — Gson deserializa el array JSON a List<Tarea>. La coroutine reanuda en el hilo principal.
  • 7
    tareas = resultado — El estado cambia. Compose detecta el cambio y re-renderiza la LazyColumn con todas las tarjetas.
  • 8
    isLoading = false — El spinner desaparece. La UI muestra la lista completa.

Crear una tarea nueva

  • 1
    Usuario escribe titulo y descripcion en los OutlinedTextField. El estado titulo y descripcion se actualiza en tiempo real.
  • 2
    Toca el boton Agregar. Se valida que el titulo no este en blanco.
  • 3
    scope.launch inicia una coroutine. Llama a api.createTarea(Tarea(...)) que hace POST /tareas con JSON en el body.
  • 4
    La API responde con la tarea creada (incluyendo el id asignado por la BD).
  • 5
    Se llama cargarTareas() para refrescar la lista. Los campos se limpian.

Marcar como completada (toggle)

  • 1
    Usuario toca el Checkbox de un TareaItem. Se dispara el callback onToggle.
  • 2
    Se crea una copia de la tarea con tarea.copy(completadaInt = if (tarea.completada) 0 else 1). Esto invierte el estado.
  • 3
    api.updateTarea(tarea.id!!, actualizada) hace PUT /tareas/{id} enviando el objeto actualizado.
  • 4
    Se refresca la lista con cargarTareas().

Endpoints de la API

La app consume una API REST con los siguientes endpoints. El servidor corre en http://localhost:3000 (o 10.0.2.2:3000 desde el emulador).

Metodo Ruta Funcion Kotlin Descripcion
GET /tareas getTareas() Retorna todas las tareas como array JSON
GET /tareas/{id} getTarea(id) Retorna una tarea especifica por ID
POST /tareas createTarea(tarea) Crea una nueva tarea. Body: JSON con titulo y descripcion

Manejo de Errores y Debug

La app tiene manejo de errores en todas las operaciones y logging HTTP para facilitar el debug.

Manejo de errores

Try/Catch en cada operacion

Cada llamada a la API esta envuelta en try/catch. Los errores se muestran al usuario con Toast.

try {
    tareas = api.getTareas()
} catch (e: Exception) {
    // Error de red, timeout, 4xx, 5xx
    Toast.makeText(context,
        "Error: ${e.message}",
        Toast.LENGTH_LONG
    ).show()
} finally {
    isLoading = false   // siempre oculta spinner
}
Logging HTTP

HttpLoggingInterceptor

Con Level.BODY muestra en Logcat el request y response completos, incluyendo headers y body JSON.

val logger = HttpLoggingInterceptor().apply {
    level = HttpLoggingInterceptor.Level.BODY
}
// Logcat → filtrar por "OkHttp"
// D/OkHttp: --> GET http://10.0.2.2:3000/tareas
// D/OkHttp: <-- 200 OK (120ms)
// D/OkHttp: [{"id":1,"titulo":...}]

Causas comunes de error y solucion

ErrorCausaSolucion
ConnectException El servidor no esta corriendo Iniciar el servidor Node.js en el host (node index.js)
CLEARTEXT not permitted Falta usesCleartextTraffic Agregar en AndroidManifest: android:usesCleartextTraffic="true"
Lista vacia / no conecta IP incorrecta en emulador Usar 10.0.2.2 (no localhost ni 127.0.0.1)
JsonSyntaxException JSON no coincide con data class Verificar nombres de campos con @SerializedName
404 Not Found BASE_URL incorrecta BASE_URL debe terminar en / y NO incluir "tareas"

Tema Visual — Material 3

La app usa Material 3 con color dinamico (Dynamic Color) disponible en Android 12+.

ui/theme/Theme.kt

AppApiTheme

Detecta si el sistema esta en modo oscuro y si el dispositivo soporta colores dinamicos (Monet, Android 12+).

@Composable
fun AppApiTheme(
    darkTheme: Boolean = isSystemInDarkTheme(),
    dynamicColor: Boolean = true,         // Android 12+ toma colores del wallpaper
    content: @Composable () -> Unit
) {
    val colorScheme = when {
        dynamicColor && Build.VERSION.SDK_INT >= Build.VERSION_CODES.S -> {
            if (darkTheme) dynamicDarkColorScheme(context)
            else           dynamicLightColorScheme(context)
        }
        darkTheme -> DarkColorScheme
        else      -> LightColorScheme
    }
    MaterialTheme(colorScheme = colorScheme, typography = Typography, content = content)
}
🌟

En Android 12+, el color primario de la app se adapta automaticamente al color del fondo de pantalla del usuario (sistema Monet de Google). No requiere cambios de codigo.

🌑

En Android 11 y anteriores, se usa el esquema de colores estatico definido en Color.kt (purpura como primario, siguiendo las guias de Material You).

  1. Android Studio (Hedgehog o superior)
